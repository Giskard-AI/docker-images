# docker-images

This repository builds and publishes Giskard-maintained Python Docker images.

## Python Images

Images are published to `quay.io/giskard/python`.

Supported Python versions are `3.12`, `3.13`, and `3.14`. For each version, the
publish workflow creates:

- default minor tags, for example `3.13`, based on Ubuntu `24.04`
- default patch tags, for example `3.13.13`, based on Ubuntu `24.04`
- default platform tags, for example `3.13-ubuntu-latest` and
  `3.13-runner-ubuntu24-arm64`
- Ubuntu-specific minor tags, for example `3.13-ubuntu24.04` and
  `3.13-ubuntu26.04`
- Ubuntu-specific patch tags, for example `3.13.13-ubuntu24.04` and
  `3.13.13-ubuntu26.04`
- Ubuntu-specific platform tags, for example
  `3.13-ubuntu26.04-ubuntu-latest` and
  `3.13-ubuntu26.04-runner-ubuntu24-arm64`

Dockerfiles are generated by `python/generate_dockerfile.sh` from
`docker-library/python` `slim-trixie` fragments. Ubuntu base image digests are
tracked in `python/base-images.json`; the generated Dockerfiles default to the
configured Ubuntu `24.04` image, while the publish workflow passes explicit base
images for each Ubuntu variant.

## Update Flow

The scheduled `Update python dockerfile` workflow refreshes the pinned
`ubuntu:24.04` and `ubuntu:26.04` digests in `python/base-images.json`,
regenerates Python Dockerfiles, then opens or updates the `cron-python-update`
pull request.

After generated Dockerfile changes land on `main`, the `Publish the images`
workflow builds the `3.12`, `3.13`, and `3.14` images for both configured
Ubuntu bases and both configured platform runners, then publishes
multi-platform manifests to Quay. Existing non-Ubuntu-specific tags continue to
point at the Ubuntu `24.04` images for compatibility.
